Welcome to AuthForge

Inspect, modify, and replay every authentication artifact your browser sees — cookies, web storage, IndexedDB, JWTs, and live network traffic — from one keyboard-friendly UI.

Take the tour
Version 1.0.0 · No telemetry · all state local · open source
AuthForge unifies what would otherwise take six separate tools — DevTools' Application tab, a JWT debugger, a request re-issuer, a network sniffer, a JWT attack generator, and a cookie editor — into one coherent workspace.

What you can do

🍪

Cookies

See every cookie across every site, search, edit values in place, and toggle flags. JWT-shaped cookies decode inline; CHIPS (partitioned) cookies are flagged and round-trip correctly. Six encoding helpers above the value field: base64, URL, JSON.

💾

Local & Session Storage

CRUD for both web-storage APIs, with refresh-token-shaped key detection. In "All open tabs" mode, entries from every open tab are merged with origin labels — see at a glance which site is storing what.

🗃️

IndexedDB editor

The hardest store to inspect through normal browser tools. List databases, expand records to edit JSON, add new records, delete safely. Embedded JWTs (like MSAL cache entries) get an inline decoder + re-signer that patches the new token back at the right JSON path.

🌐

Network capture

Passive observer of HTTP(S) headers via the chrome.webRequest API. Captures Authorization headers, Set-Cookie responses, custom auth headers, token-endpoint requests. Optional deep capture via chrome.debugger reads response bodies too — opt-in, per tab.

⚒️

Request Workbench

Expand any captured request and edit everything: method, URL, query params, headers, body, credentials, redirect mode, referrer, integrity, keepalive, timeout. Send and see the response inline. Repeat N times for race-condition probes. Round-trips with curl: Copy as curl / Import curl.

🔑

JWT toolkit

Decode every JWT to its claims. Edit and re-sign with HMAC, or generate attack variants categorised as recon, test, or attack (alg=none, alg confusion, kid injection, claim tampering). Replay each variant against the original endpoint. HMAC brute-force for HS256/384/512.

🟦

Microsoft Entra / FOCI

For Entra (Azure AD) tokens specifically: identifies audience, scope, tenant, FOCI family membership. Authenticated recon against 26 Microsoft endpoints across 5 audience groups. FOCI refresh-token exchange with 13 pre-filled first-party clients for lateral pivots.

🛡️

Audit

Role-aware audit: cookies are classified as auth / csrf / analytics / functional, and findings tailored to the class (an analytics cookie without HttpOnly is informational, an auth cookie without HttpOnly is high severity). Every finding shows the source domain. JSON export for reports.

📸

Snapshots & Profiles

Capture the full storage state of a site and restore it later. Profiles bundle cookies + storage and can be transplanted across domains with optional remapping (e.g. dev.example.com → staging.example.com). Import / export as JSON.

Common workflows

I want to…

Decode a JWT I see in localStorage. Click the Tokens tab — every JWT-shaped value across cookies, storage, network, and IndexedDB is collected with one source per row. Click any token to decode.

I want to…

Replay a request with a different bearer token. Network tab → expand the captured request → click 🔑 Use captured token. Pick from any detected token; the Authorization header swaps. Hit Send.

I want to…

Check what a stolen refresh token can reach. Tokens tab → click the RT → FOCI exchange. Pre-fills with 13 known first-party Microsoft client IDs. Pick one and you'll see an access token for that audience.

I want to…

Audit a multi-tenant site for cookie security. Scope picker → All open tabs → Audit tab. Every finding is labelled with its source domain so you can export the JSON and pivot the report by host.

I want to…

Save my dev session before clearing storage. Snapshots tab → "Capture current state". Restore later from the same tab. Profiles let you do the same across domains with optional cookie-domain remapping.

I want to…

Brute-force an HS256 JWT signing secret. Tokens tab → click the JWT → JWT toolkit → Brute-force HMAC secret. Default top-1000 wordlist or paste your own. Hits show the cracked secret inline.

Permissions, explained

AuthForge requests the following permissions. Each one is justified by a specific feature; nothing is asked for "just in case". The full source is readable and every privileged operation goes through a single dispatcher in background/service-worker.js.

cookies

Read, create, modify, delete cookies. The whole Cookies tab.

storage

Persist settings, snapshots, profiles via chrome.storage. All local.

tabs

Identify which tab to operate on. Required for current-tab and all-tabs scopes.

activeTab

Temporary scripting access to the tab you opened AuthForge on.

scripting

Inject reader scripts into the page MAIN world for LS / SS / IDB. Also: Workbench replay.

sidePanel

Alternative UI surface for users who prefer the side panel over the toolbar popup.

webRequest

Observe-only HTTP(S) headers. No blocking, no modification. The Network tab needs this.

debugger

Opt-in per tab. Reads response bodies via CDP. Shows Chrome's yellow "debugging" banner.

host_permissions: <all_urls>

Cross-site analysis tool — AuthForge needs to operate on whichever site you're currently inspecting. It does not contact any remote endpoint under the author's control. Cross-domain actions are user-initiated only. The Microsoft Graph recon feature talks to Microsoft-owned hosts only via a static allowlist in the service worker.

Privacy in one sentence

AuthForge runs entirely in your browser, contacts no remote endpoint under the author's control, has no analytics or telemetry, and stores nothing outside chrome.storage.local (your machine, your profile).
