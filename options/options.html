Advanced

User interface

Hide empty storage tabs
Hide Local / Session / IndexedDB tabs in the popup when they have no entries. Most sites only use one of these — hiding the empty ones keeps the tab bar uncluttered. Turn off to always see every tab.
Compact mode
Tighter row spacing — fits more entries on screen at the cost of a busier look.

Network capture

Auto-start recording
Network capture begins automatically when a new browser session starts. Off keeps it paused until you click Start.
Buffer size
Maximum number of auth-relevant entries to keep in memory. Older entries are dropped first.
Persist captures across browser sessions
⚠ Stores network captures in chrome.storage.local (which survives browser restart) instead of chrome.storage.session (wiped on browser close). Captures contain raw tokens — only enable if you're working on a single-user device and want your last-session data available later.

Security testing

Merged with the ~50 baked-in defaults. Useful for adding secrets you recovered from leaked source or env files.
Max brute-force attempts
Caps the number of candidates tried in one run. The default is plenty for built-in wordlists; raise for big custom dictionaries.

Data management

Clear network capture buffer
Removes every captured request. AuthForge starts fresh on the next request.
Clear all snapshots
Removes every saved domain snapshot. Profiles are kept.
Clear all profiles
Removes every saved credential profile. Snapshots are kept.
Export everything (JSON)
All settings + profiles + snapshots in one JSON file. Sharable; re-importable with the matching button.
Import everything (JSON)
Restores from a previous Export All file. Existing entries with the same id are overwritten.

Profiles

Named, reusable credential bundles for cross-environment testing. Capture the auth state of one site, apply it to another (with domain remapping), or import from a teammate's HAR file. Export to cURL, Postman, HTTPie, or .env for use in your test pipelines.

Profile

Cookies will be scoped to this URL. Defaults to the active tab.
Advanced — raw JSON
Editing the raw bundle directly. Be careful: this overwrites the profile on save.

Snapshots

Saved domain states — cookies, localStorage, sessionStorage and IndexedDB databases — captured from the popup or DevTools panel. Restore them from the same surfaces.

About

Capabilities

  • Full CRUD for cookies on any URL, with every attribute exposed (Domain, Path, SameSite, Secure, HttpOnly, host-only, expiration).
  • Read and edit the active page's localStorage, sessionStorage, and IndexedDB records — including binary-safe serialization of Date, Blob, and ArrayBuffer.
  • JWT detection across all storage with a one-click decoder, plus a standalone HS256/384/512 signer in this page.
  • Refresh-token heuristic flag any key whose name looks like a long-lived credential.
  • Domain snapshots: capture the full state of a site in one click, restore later, export as JSON.
  • Undo / redo on every destructive edit during a session (Ctrl+Z, Ctrl+Shift+Z).
  • Import / export as JSON, Netscape cookies.txt, raw Cookie: header, or a ready-to-paste cURL command.

Keyboard shortcuts

  • Ctrl+F — focus search
  • Ctrl+Z / Ctrl+Shift+Z — undo / redo
  • R — refresh current tab's data

Privacy

AuthForge runs entirely in your browser. No data is sent to any server. The HMAC signing in the JWT playground uses the Web Crypto API locally.
